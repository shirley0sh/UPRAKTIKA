# Элементы теории графов

**Учебный веб-проект**  
**Команда №7**: Голубев, Матвеева, Сухоруков, Никифоров

## О проекте

Веб-приложение реализует ключевые алгоритмы теории графов для неориентированных графов:
- поиск минимального остовного дерева;
- поиск кратчайших путей.

Приложение позволяет интерактивно задавать граф через матрицу расстояний (весов) и получать результаты работы четырёх классических алгоритмов.

**Реализованные алгоритмы:**
1. Алгоритм Прима (минимальное остовное дерево)
2. Алгоритм Дейкстры (кратчайшие расстояния от заданной вершины)
3. Алгоритм Флойда–Уоршелла (матрица кратчайших путей между всеми парами вершин + матрица маршрутов)
4. Алгоритм Краскала (минимальное остовное дерево)

Входные данные
Количество вершин (n) — целое число от 2 до 20.
Матрица весов — квадратная матрица размера n × n.
0 на главной диагонали.
∞ или большое число (например, 9999) при отсутствии ребра.
Симметричная матрица (для неориентированного графа).
Для алгоритма Дейкстры дополнительно указывается вершина-источник (индексация с 0).

Реализованные алгоритмы

1. Алгоритм Прима
Находит минимальное остовное дерево (МОД) графа, начиная с произвольной вершины и постепенно добавляя ребро наименьшего веса, которое соединяет дерево с новой вершиной.
Результат: список рёбер МОД и его общая стоимость.
2. Алгоритм Дейкстры
Находит кратчайшие расстояния от заданной вершины-источника до всех остальных вершин в графе с неотрицательными весами.
Результат: расстояния и пути до каждой вершины.
3. Алгоритм Флойда–Уоршелла
Вычисляет кратчайшие пути между всеми парами вершин. Также строит матрицу маршрутов для восстановления самих путей.
Результат:Матрица кратчайших расстояний, матрица следующей вершины (для восстановления пути)
4. Алгоритм Краскала
Находит минимальное остовное дерево, сортируя все рёбра по весу и добавляя их, если они не образуют цикл (используется Union-Find).
Результат: список рёбер МОД и его общая стоимость.

Примечания

Приложение проверяет корректность ввода: соответствие размера матрицы n, наличие главной диагонали равной 0, симметричность.
При ошибках ввода (неполная матрица, неверный формат чисел, отрицательные веса для Дейкстры и т.д.) выводятся понятные сообщения об ошибке.
Алгоритмы работают только с неориентированными графами.
Для больших n (>15) интерфейс может замедляться из-за визуализации.
